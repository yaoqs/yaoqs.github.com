Virtual PC 2007 & Debian Linux | Virtual Positiveness

Yao Qing-sheng
Department of Natural Philosophy & Infomation Sciences, CHINA
Abstract Keywords Reprint , Draft , 虚拟机 , Virtual PC ,
Citation Yao Qing-sheng.  Virtual PC 2007 & Debian Linux | Virtual Positiveness.  FUTURE  Natural Philosophy & Infomation Sciences,  2019:1127,  https://yaoqs.github.io/2019/11/27/Virtual%20PC%202007%20&%20Debian%20Linux%20Virtual%20Positiveness/

I recently had to install a fresh new Debian Virtual Machine using Microsoft Virtual PC 2007 (free). Officially, only Windows 98, ME, 2000, XP, Vista, 2008, OS/2 are supported. But of course, users are able to install almost any Linux distribution as guest operating system, as long as they’re motivated enough to spend a long time reading message boards & posts to find out why it just doesn’t work. In this post, I’ll focus on :

How to install Debian Linux on Virtual PC
How to install “Virtual Machine Additions for Linux” (and why you should do so)
Pre-requisites: what to do first

This is not a beginner’s guide to VMs / Linux, thus I’ll go quickly through the first steps and through the installation of Debian itself, but don’t hesitate to ask questions in the comments.

Download a Debian CD image (i386). No need to burn it.
Download and install Microsoft Virtual PC 2007 SP1
Download and install Microsoft VM Additions for Linux (yes, it’s designed for Virtual Server and not Virtual PC)
Open Microsoft Virtual PC and create a new Virtual Machine. 64 or 128 MB of memory will be enough.
How to install Debian

Start the Virtual Machine
CD > Capture ISO Image
Choose your Debian CD image (ISO file)
Reboot your Virtual Machine
You’ll see a beautiful Debian logo and a prompt asking you to “press enter to install Debian“. Yep. But it didn’t work for me. Sometimes, the installer hangs in the first steps, sometimes after displaying the GUI, sometimes after the drive check. What worked for me was to run the installer with this command (instead of simply pressing enter):

install noapic nolapic clock=pit

A really simple explanation of noapic nolapic is available here. According to Microsoft (yep!),

After you install a Linux-based virtual machine in Microsoft Virtual Server 2005 R2, the system time in the Linux guest operating system runs too fast. For example, the current time in the Linux guest operating system may advance by one minute approximately every 48 seconds. The clock=pit parameter causes the Linux 2.6 kernel to use a more efficient algorithm to synchronize time between the virtual machine and the host computer. This algorithm does not adjust for lost ticks. Therefore, the algorithm does not cause the virtual machine to gain time.

In my case, clock=pit didn’t only allow my VM to have a correct clock, but allowed the installer to load properly (!).

Now, you just have to follow every step to install your Debian, as on a real machine ( need more details?).

No graphical interface – How to configure X

Installation OK, first boot… Black screen, or unreadable text. In fact, everything but the graphical interface (X) works correctly. X is set to display a color depth of 24 bits, but Virtual PC only supports 16 or 32 bits. Now what?

Stay on this messed screen and press CTRL + ALT + F6

This will open a prompt : “GNU/Linux tty6 – login:”
Type “root” and the password you set during installation
At the prompt (yourcomputer:~#), type “nano /etc/X11/xorg.conf“
Search for “DefaultDepth” (CTRL + W)
Replace “24″ by “16″
Below, there’s again a “Depth” set to “24″ – replace it by “16″
Save your changes (CTRL + O) and exit nano (CTRL + X)
Reboot your graphical interface (CTRL + ALT + BACKSPACE) or reboot your virtual machine
It should now work!
Virtual Machine Additions for Linux – Worth a try

Microsoft officially supports Linux (at least SuSE and Red Hat) as guest operating system on Microsoft Virtual Server. Here, we’re using Microsoft Virtual PC, but the VM Additions Microsoft provides also work on Virtual PC.

Why should we bother installing them? Two main reasons:

Time Synchronization. Useful with “Saved State” mode.
Mouse support. When the mouse reaches the edges of the VM, the mouse is released from the VM, making it really more convenient to use and avoiding unnecessary uses of the ALT key (VM users know what I’m talking about!)
To install them:

CD > Capture ISO Image
Capture the ISO image “VMAdditionsForLinux.iso” you installed (probably in C:\Program Files\Microsoft Virtual PC\Virtual Machine Additions)
Open a shell as root (type “su“)
Find out your kernel version (type “uname -r“)
Install the following packages: alien, build-essential, chkconfig, fileutils, linux-headers-2.x.xx-xx-386, linux-image-2.x.xx-xx-386, linux-source-2.x.xx (you can do it by typing “apt-get install” followed by the names of these packages. For chkconfig, take a look here)
Change to your CDROM’s directory (cd /media/cdrom)
Install the main package by typing:
rpm -ivh vmadd-full-2.0-1.i386.rpm –force
–nodeps
It didn’t work for me – you can also install each package one by one, especially “full-RHEL”, “x11″ and “timesync”
To run them (I’ll use the example of vmadd-timesync):

Type “ls /etc/init.d/vmadd*” to see the additions you installed
Add them one by one to your “autostart” list with the help of update-rc.d:
update-rc.d vmadd-timesync defaults
To get this wonderful mouse support, one more step is required:

Type

Restart X (CTRL + ALT + BACKSPACE) or reboot your VM
That’s all! Up and running, and ready for all the security-related tests you want to do with it.
